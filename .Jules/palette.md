## 2024-10-24 - Accessibility Fixes for Static Pages
**Learning:** Found that custom visual styles often strip out native focus indicators, making keyboard navigation difficult. Also, decorative elements acting as links (like the lantern) need explicit ARIA labels to be meaningful to screen readers.
**Action:** Always verify keyboard focus state (`:focus-visible`) when overriding default link styles, and always add `aria-label` to icon-only interactive elements. Use `aria-current` for navigation states.
## 2026-05-02 - Dual-Labeling and Iframe Accessibility Context
**Learning:** Found that while `aria-label` provides essential context for screen readers on icon-only interactive elements (like the lantern link), sighted users may also need explicit context. Dual-labeling using both `aria-label` and `title` attributes ensures screen reader accessibility while simultaneously providing native tooltips for sighted users. Additionally, `iframe` elements used for external media (like YouTube videos) lack context without a descriptive `title` attribute, which screen readers rely on to announce the iframe's content.
**Action:** Always implement dual-labeling (`aria-label` + `title`) on icon-only interactive elements to serve both assistive technology and sighted users. Ensure `iframe` elements include descriptive `title` attributes to establish context before the embedded content is interacted with.

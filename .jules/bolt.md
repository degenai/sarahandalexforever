## 2026-06-13 - Date Coercion Overhead in Repeated Calculations
**Learning:** In Javascript, repeatedly calculating the difference between a `Date` object and a timestamp (e.g., `targetDate - Date.now()`) within a high-frequency loop, like a `setInterval` or `requestAnimationFrame`, incurs performance overhead due to implicit type coercion. When a mathematical operation happens, the engine must convert the `Date` object to a primitive number every single time. As measured locally, caching the timestamp using `.getTime()` reduces the execution time of this check by around 60%.
**Action:** When a constant `Date` object is used in mathematical operations, especially within high-frequency intervals or loops, always cache its primitive numeric value via `Date.getTime()` beforehand to bypass implicit type coercion.

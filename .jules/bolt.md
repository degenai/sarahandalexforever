## 2024-06-07 - Caching DOM elements and avoiding implicit coercion in requestAnimationFrame/setInterval loops
**Learning:** In the countdown functions within `desktop.html` and `mobile.html`, the `setInterval` ticker was calling `document.getElementById` and invoking implicit type coercion via `Date.now() - new Date('...')` on every single tick (every 1 second). This causes unnecessary overhead over time. Benchmarks showed caching the elements and using `.getTime()` explicitly on the `target` Date improved performance of the `tick()` function by ~84%.
**Action:** When implementing high-frequency DOM update loops (like countdowns, scroll handlers, or animations), always cache `document.getElementById` or `document.querySelector` outside the loop, and store explicit `.getTime()` numeric values instead of raw Date objects to avoid coercion overhead.
